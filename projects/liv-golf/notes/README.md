# Notes Repository

A portable markdown-based notes system. Works with Obsidian, Notion, Bear, Logseq, Typora, OneNote, GitHub, and any plain text editor.

## Quick Links

- 📁 [Projects](#projects-index) — living status docs for each workstream
- 👤 [People](people/README.md) — 1:1 and relationship notes

## Structure

```
notes/
├── README.md               ← This file (index & conventions)
├── meetings/
│   └── YYYY/
│       └── YYYY-MM-DD-title-slug.md
├── projects/               ← Per-project status & history docs
├── people/                 ← Per-person 1:1 and relationship notes
├── inbox/                  ← Drop zone for emails, attachments, offline notes
└── reference/              ← (future) evergreen reference docs
```

## File Naming Convention

- **Meetings:** `YYYY-MM-DD-short-descriptive-title.md`
- **Projects:** `projects/project-name.md`
- Use lowercase and hyphens, no spaces

## Note Template

Each meeting note includes:
- Title, date, attendees, and source link at the top
- Sections for context, key decisions, and action items
- A checkbox list (`- [ ]`) for action items so they render interactively in most apps
