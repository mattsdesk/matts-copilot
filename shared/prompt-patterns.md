# Prompt Patterns

Reusable prompt fragments for common tasks. Copy, adapt, and use these as starting points.

---

## Research

### Summarize a source
```
Summarize the key points from [source/text]. Focus on:
- Main argument or finding
- Supporting evidence
- Implications or so-what
Keep it under [X] words. No filler.
```

### Comparative analysis
```
Compare [A] and [B] across these dimensions: [dim1], [dim2], [dim3].
Present as a table. Add a 2-sentence recommendation at the end.
```

### Research brief
```
I need a research brief on [topic] for [purpose/audience].
Cover: background, key players, current state, open questions.
Use headings. Keep each section to 3-5 bullets.
```

## Writing & Content

### First draft from brief
```
Write a first draft of [content type] based on the brief below.
Tone: [tone reference or describe]
Length: [word count or rough length]
Format: [headers / no headers / bullets / prose]

Brief:
[paste brief]
```

### Improve existing draft
```
Improve this draft. Make it tighter and more direct. Cut anything that doesn't add value.
Don't change the core meaning or structure — just sharpen it.

Draft:
[paste draft]
```

### Rewrite for different audience
```
Rewrite the following for [audience]. Keep the substance but adjust the tone and vocabulary.

Original:
[paste content]
```

## Project & Client Work

### Project kickoff brief
```
Based on the context below, write a project brief that covers:
- Objective (1-2 sentences)
- Scope (what's in, what's out)
- Deliverables (list)
- Success criteria
- Key assumptions

Context:
[paste context]
```

### Decision log entry
```
Summarize this decision for the project memory log:
- What was decided
- Why (key reason)
- Who was involved
- Date: [date]
```

### Status update
```
Write a brief project status update for [audience: client / team / stakeholder].
Tone: [professional / casual]
Cover: what's been done, what's next, any blockers.

Context:
[paste notes]
```

## Editing & QA

### Proofread
```
Proofread the following. Flag grammar issues, unclear sentences, and inconsistent tone.
Don't rewrite — just annotate what to fix and why.
```

### QA against brief
```
Review this output against the brief below. Note anything missing, off-brief, or unclear.
Score it: [Ready to send / Needs minor edits / Needs major revision]

Brief:
[paste brief]

Output:
[paste output]
```

## Memory & Summarization

### End-of-session memory update
```
Based on our session today, write a memory log entry for [project].
Include:
- What was worked on
- Key decisions made
- Any approved language or locked-in choices
- Open items or next steps
Format for appending to memory.md.
```
